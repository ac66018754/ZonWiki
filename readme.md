此 Repo 用來存放我個人的筆記，
所以叫做 ZonWiki。

想解決的問題:
1. 筆記過於散亂，存放在記事本、LINE KEEP、Notion、OneNote等地，應該要集中一處
2. 有時候用 google 搜尋或是問 AI 得到的答案我們看過就忘，應該要整理成筆記來達成長期記憶
3. 承上，有些人會直接從網頁把搜到的或問到的東西複製下來，但這樣格式可能會跑掉，或是很醜，難以閱讀，也沒時間好好整理

用法:
1. 當你遇到一個問題，用 google 搜尋或是問 AI 得到了答案
2. if 你知道要把結果放在哪裡，看需不需要新增資料夾、檔案，或是用既有的就好，自己貼上 (此時可能醜醜的、亂亂的)
3. then 執行 ./腳本/format-md.ps1，會調用 gemini api 對剛異動的內容進行美化編排
4. else 若還不知道要把結果放在哪裡，或沒有想法，則先在 ./筆記區/raw 新增一個檔案，然後直接貼上
5. then 執行 ./腳本/整理raw專區.ps1(先暫時不寫)，讓 AI 來決定要這份檔案內的資料寫進哪些資料夾的哪些檔案 (可能一個，也可能多個)
6. 執行 ./auto-commit.ps1 可以自動 commit % push 到 GitHub 上

資料夾層級
1. ./.github: 用來放 github action 設定文件
2. ./關於此Repo的計畫: 用來放對於此 Repo 的規劃、想法
3. ./腳本: 用來放此 Repo 專用的腳本
4. ./筆記區/raw: 用來放你不知道要如何處理的資料，會讓 AI 來判斷該將這些資料收整自哪些資料夾的哪些檔案。或是放突然想到的問題
