# 環境變動清單

> 這份文件記錄為了開發 ZonWiki MVP，**對你電腦做了哪些變動**。
> 你可以用它來：
> 1. 知道目前有什麼東西還在跑
> 2. 如果不想要這個專案了，知道要刪除/停止什麼
> 3. 如果之後想重新啟動，知道要做什麼
>
> 撰寫時間：2026-04-15

## TL;DR

**我安裝過的「系統級」軟體：零個。** 你的 Node、.NET、pnpm、Docker、PostgreSQL 都是原本就有的。

**我做過的事情可分四類**：

1. **在 repo 內**新增了 .NET solution、前端專案、設定檔、文件
2. **在 Docker** 拉了 postgres:16-alpine 映像、建了一個容器和 volume
3. **在套件快取**（NuGet 全域快取、pnpm 全域 store、frontend/node_modules）多了一堆套件
4. **在 PATH** 暫時切換 Node 18 → 20（**僅該次終端機 session，沒有改寫系統設定**）

## 2. Docker 多了什麼

| 類型 | 名稱 | 大小 / 用途 |
|---|---|---|
| 映像 | `postgres:16-alpine` | 約 395 MB（從 Docker Hub 拉的） |

| 網路 | `zonwiki_default` | docker compose 自動建的內部網路 |

## 4. dotnet-ef 工具

- 我用 `dotnet new tool-manifest` 建了 `dotnet-tools.json`（**只在 ZonWiki 這個 repo 內**）
- 然後 `dotnet tool install dotnet-ef --version 10.0.0`（**也是 local tool，不是 global**）
- 這代表它**只屬於這個專案**，不會影響其他 .NET 專案，也不會留在系統 PATH

## 6. Node 版本切換

- 你電腦上原本就有 nvm-windows，裡面有三個版本：v18.4.0、v18.18.0、v20.12.2
- Next.js 16 要求 Node ≥ 20，所以執行前端時我用了 v20.12.2
- 我的做法是**在當下的 bash session 裡 export PATH**，並沒有：
  - 改寫系統 PATH
  - 跑 `nvm use` 改變預設版本
  - 安裝任何新的 Node 版本

> 也就是說，你開新終端機的話，`node --version` 仍然會是你原本的預設版（v18.18.0）。

## 如何「**暫時停止**」開發環境（最常見）

只是不想讓 Postgres 一直跑、之後還會回來開發：

```bash
cd D:/Repos/SideProjects/ZonWiki
docker compose stop
```

> Volume 與資料**會保留**，下次 `docker compose start` 直接從停止的地方繼續。

要再次啟動：

```bash
docker compose start
```

## 如何「**完整重啟**」整個網頁版

讓網頁完全可用（Postgres + API + 前端）：

```bash
# 終端機 1：Postgres
cd D:/Repos/SideProjects/ZonWiki
docker compose up -d
# → 確認：docker compose ps 應該看到 healthy

# 終端機 2：後端 API（會自動套 migrations + 第一次 sync）
cd D:/Repos/SideProjects/ZonWiki
dotnet run --project src/ZonWiki.Api --launch-profile http
# → http://localhost:5009/healthz 回 "Healthy"
# → 不要關這個終端

# 終端機 3：前端（Node 必須 ≥ 20）
cd D:/Repos/SideProjects/ZonWiki/frontend
# 如果 node --version 顯示 18.x，先切到 20：
#   PowerShell: nvm use 20.12.2
#   bash:       export PATH="/c/Users/User/AppData/Roaming/nvm/v20.12.2:$PATH"
pnpm run dev
# → http://localhost:3000
```

## 如何「**完全刪除這個專案的環境痕跡**」

依風險由小到大列：

### 步驟 A：停止並刪除 Docker 容器 + volume（含資料庫資料）

```bash
cd D:/Repos/SideProjects/ZonWiki
docker compose down -v
```

這會：
- 停止並移除 `zonwiki-postgres` 容器
- 移除 `zonwiki_default` 網路
- 移除 `zonwiki-postgres-data` volume（**裡面的筆記索引也會一起消失**，但筆記原檔在 `筆記區/` 不受影響）

### 步驟 B：刪除 Docker 映像（選擇性，省約 395 MB）

```bash
docker image rm postgres:16-alpine
```

只有當你確定**沒有別的專案用到它**才刪。

### 步驟 C：刪除 node_modules 和 .NET build 產物（選擇性，省幾百 MB）

```bash
cd D:/Repos/SideProjects/ZonWiki
rm -rf frontend/node_modules
rm -rf frontend/.next
find . -type d -name bin -exec rm -rf {} +
find . -type d -name obj -exec rm -rf {} +
```

### 步驟 D：清理 NuGet 全域快取（**不建議，會影響其他 .NET 專案**）

```bash
dotnet nuget locals all --clear
```

只有在你完全不再寫 .NET 才做。其他專案下次 build 會自動重抓。

### 步驟 E：刪除整個 Repo

```bash
# ⚠️ 注意：你的筆記原檔（筆記區/）也在這個 Repo 裡，刪掉 Repo 等於刪掉筆記
# 如果只是不想要網頁版，建議只做步驟 A~C
```

## 風險與注意事項

| 項目 | 說明 |
|---|---|
| Postgres volume | 裡面有同步進去的筆記索引和留言。**筆記原檔不在這裡**，在 `筆記區/`。volume 刪了不會弄丟筆記 |
| 5433 port 衝突 | 我選 5433 是因為你 5432 已被本機 Postgres 佔用。若 5433 之後也被佔，改 `docker-compose.yml` 與 `appsettings.Development.json` 同步調整 |
| 開機自動啟動 | `docker-compose.yml` 寫了 `restart: unless-stopped`，意思是：開機後 Docker Desktop 啟動時，這個容器會自動跑。如果不想要，把那行改成 `no` 或執行 `docker compose stop` 一次（`stop` 後會記住停止狀態） |
| dotnet-tools.json | 只在這個 repo 生效。如果你刪掉 repo，這個工具自然也消失 |

## 後續若想完全乾淨地重灌

1. `docker compose down -v` 在 Repo 內
2. `docker image rm postgres:16-alpine`
3. 砍 `frontend/node_modules` 和所有 `bin/`、`obj/`
4. （選擇性）`dotnet nuget locals all --clear`
5. 砍 Repo 資料夾（**先確認 `筆記區/` 已備份**）

完成後你的電腦會回到開發 ZonWiki 之前的狀態，差別只剩：
- pnpm 全域 store 還有一些快取（無害，下次裝其他 Node 專案時可重用）
- Docker 還記得它曾經拉過 postgres:16-alpine 映像（如果步驟 2 沒做）
