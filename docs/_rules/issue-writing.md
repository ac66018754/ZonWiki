# ISSUE WRITING RULES

When encountering ANY error, you MUST create an issue record.

## File Location
/docs/issues/YYYY-MM-DD-<short-name>.md

## Required Structure

# ISSUE: <title>

## Problem
What happened

## Context
System state, input, environment

## Root Cause
Actual cause (NOT symptom)

## Investigation Process
Step-by-step debugging process

## Solution
How it was fixed

## Impact
- Affected components
- User impact

## Time Cost
Time spent

## Prevention
How to avoid in future

## Related Files
List of files involved
