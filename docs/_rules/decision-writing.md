# DECISION WRITING RULES

When making ANY architectural or technical decision, record it.

## File Location
/docs/decisions/YYYY-MM-DD-<topic>.md

## Structure

# DECISION: <title>

## Context
Why decision is needed

## Options Considered
- Option A
- Option B

## Decision
Chosen option

## Reason
Why this option

## Consequences
- Pros
- Cons

## Rules (IMPORTANT)
Convert decision into rules:

- MUST use ...
- MUST NOT use ...
