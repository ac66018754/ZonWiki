## 2. 目標資料模型（草案，遵 `{Table}_{Field}` + 6 審計欄 + ValidFlag 軟刪除）

> 既有 ZonWiki DB 內無珍貴資料（筆記可重匯），故 schema **重新設計**；開問啦 DB 有珍貴資料，走「遷移」。
> 時間欄一律 UTC。內容欄一律 UTF-8。

### 2.1 身分與租戶
- **`User`**（沿用 + 擴充）：`GoogleSub / Email / DisplayName / AvatarUrl` + 新增 `User_TimeZone`（IANA，如 Asia/Taipei）、`User_DisplayMode`（warmpaper/light/dark/night）。
- **`Workspace`**（租戶邊界）：`Workspace_Name / Workspace_OwnerUserId`。每位使用者預設一個個人 Workspace。
- **`WorkspaceMember`**：`_WorkspaceId / _UserId / _Role`（Owner/Editor/Viewer）。權限以此為準。
- 所有內容表都帶 `{Table}_WorkspaceId`，EF 全域過濾＝「目前使用者可存取的 Workspace」+ `ValidFlag`。

### 2.2 筆記（含日記）
- **`Note`**（由舊 `Article` 進化）：`_WorkspaceId / _Title / _Slug / _ContentRaw(md) / _ContentHtml(快取) / _ContentHash / _SourceFilePath(nullable，僅匯入用) / _IsDraft / _Kind(note|journal) / _JournalDate(nullable, UTC 日期)`。日記＝`Kind=journal` 的 Note（決策 13）。
- **`Category`**：`_WorkspaceId / _ParentId(階層) / _Name / _FolderPath(nullable, 匯入用)`。
- **`NoteCategory`**（M2M 關聯表，依你白板：筆記屬於多個分類，先正規化）：`_NoteId / _CategoryId`。
- **`Tag`**：`_WorkspaceId / _Name`；**`NoteTag`**：`_NoteId / _TagId`。
- **`NoteLink`**（wiki 反向連結）：`_SourceNoteId / _TargetNoteId / _AnchorText`。
- **`Comment`**（沿用，支援範圍留言）：`_NoteId / _UserId / _Content / _AnchorType(full|range) / _AnchorData(json)`。

### 2.3 任務（日程規劃 / Todo，依你白板）
- **`TaskCard`**（卡片式；避開 C# `Task` 字衝突，型別名用 `TaskCard`，**資料表名 `TaskCard`**）：`_WorkspaceId / _Title / _Content(md) / _Status(todo|doing|done|… 可自訂) / _Priority(0-3) / _PlannedDateTime(nullable,UTC) / _DueDateTime(nullable,UTC) / _GroupId(nullable) / _SortOrder / _RecurrenceRule(nullable)`。
- **`TaskGroup`**（卡片可建群組、Todo 可放群組）：`_WorkspaceId / _Name / _Color / _SortOrder`。
- **`TaskRelation`**（卡片間關聯、**無上下級**＝對等圖）：`_SourceTaskCardId / _TargetTaskCardId / _Kind(related)`。
- **`NoteTaskLink`**（筆記↔卡片 M2M）：`_NoteId / _TaskCardId`。
- 多視圖（清單/看板/行事曆）由查詢與前端組合，無需額外表。

### 2.4 首頁元件
- **`QuickLink`**（常用連結卡）：`_WorkspaceId / _UserId / _Title / _Url / _IconKey / _SortOrder`。
- **`CaptureItem`**（快速捕捉 Inbox；首頁與筆記頁共用）：`_WorkspaceId / _UserId / _Source(web|voice|text) / _RawContent / _AudioPath(nullable) / _Status(inbox|filed) / _FiledTargetType(nullable) / _FiledTargetId(nullable)`。錄音先存檔；轉文字方式列待討論。

### 2.5 開問啦（移植，依**最新**原始碼）
移植實體（確切欄位於 Phase 4 逐檔對齊）：`Canvas / CanvasCategory / CanvasSystemPrompt / Category / CategorySystemPrompt / SystemPrompt / Node / Edge / InlineLink / Highlight / NodeImage / NodeRevision / AiSession / AiMessage`。全部加上 `_WorkspaceId` 納入多租戶。
- **命名衝突**：開問啦有 `Category`，筆記也有 `Category` → 表名會撞。處理：開問啦的畫布分類改名（暫定表名 `CanvasCat` 或併入既有 `CanvasCategory` 體系），筆記的 `Category` 保留。Phase 4 對齊原始碼時定案（已記入待討論清單）。

### 2.6 AI 與設定（金鑰入 DB）
- **`AiModel`**（取代 `ai-models.json`）：`_Key / _Label / _Provider(ClaudeCli|OpenAiCompatible) / _Kind(chat|image) / _Enabled / _ModelId / _BaseUrl / _ApiKeyEncrypted / _TimeoutSeconds / _Notes`。
  - **API 金鑰加密儲存**（用 ASP.NET Core Data Protection 加密欄位，不存明碼；解析時若值為 `${ENV}` 仍支援環境變數覆寫，相容舊設定）。
  - 範圍：**全站共用（管理員維護）**，因金鑰屬基礎設施、<10 位可信使用者（已記待討論，可改 per-Workspace）。
- 顯示模式、時區：存 `User`（per-user）。垃圾桶：靠 `ValidFlag` + 新增 `{Table}_DeletedDateTime(nullable)` 支援「跨模組垃圾桶」視圖與日後自動清除。

## 4. 前端資訊架構（IA）

- **Shell**：固定 **Header** + **左側欄**。
  - Header：主功能切換（首頁/筆記/Todo/開問啦）、全域搜尋、帳號/系統設定、顯示模式切換。
  - 左側欄：當前主功能的附屬功能/情境（例：筆記頁＝分類樹/標籤；Todo 頁＝群組/視圖切換；開問啦＝畫布清單）。
- **首頁**（精簡聚合，完整功能在各頁）：當週日曆、今日 Todo、常用連結卡、快速捕捉（錄音/打字）。
- **設計標準（決策 9）**：除開問啦頁外，全部頁面用 **frontend-design skill**＋業界推薦做法，做到產品級（配色/元件/排版/字體/動效/間距）。**開問啦頁移植原樣、不動**。
- **RWD（決策 10）**：手機優先斷點；觸控（拖曳/長按）與滑鼠（hover/右鍵）並存；側欄在手機收成抽屜。
