# 任務規劃（Todo & Planning）模組設計決策

> 紀錄日期：2026-06-16。這份文件說明「日程規劃」頁這次強化（期間、子任務、分類、狀態釐清、UX 重構）背後的設計取捨，讓後人知道當初為何這麼做。

## 背景需求

使用者希望日程規劃能：(a) 設定任務「期間」（何時開始、何時截止）、(b) 把任務拆成「子任務」、(c) 幫任務「分類」（工作 / SideProject / 研究…）或貼標籤、(d) 保留 todo/doing/done 但覺得叫「群組」很奇怪、(e/f) 整體優化、要好的 UI/UX。

## 關鍵決策

### 1. 狀態 vs 群組的混淆 → 狀態＝看板欄位、群組改稱「分類」
- **現況問題**：`TaskCard.Status`（todo/doing/done）本來就是獨立欄位、看板也已用它分欄；但畫面上同時有一個「群組（TaskGroup）」管理區，使用者誤以為群組是用來放 todo/doing/done。
- **決策**：
  - **狀態（Status）** 維持為第一級欄位，看板三欄固定為 **待辦 / 進行中 / 完成**（`STATUS_META`），拖曳卡片即改狀態。
  - **群組（TaskGroup）** 在 UI 一律改稱「**分類**」，定位為「情境/專案桶」（工作、SideProject、研究…），可上色、可篩選、卡片上以彩色 chip 呈現。
- **為何不直接把資料表 `TaskGroup` 改名為 `TaskCategory`？** 改表名要連動改 11 個欄位、外鍵、索引、`TaskCard.GroupId`、所有端點/DTO/前端，且既有資料需 migration 搬移，風險與雜訊大。依「DB 不要跳來跳去」原則，**保留實體/資料表名 `TaskGroup`，僅在 UI/語意上呈現為「分類」**，並在程式碼註解標明。日後若真要改名再做。

### 2. 任務期間（a）→ 沿用既有 `PlannedDateTime` + `DueDateTime`
- 資料模型本來就有 `PlannedDateTime`（排程/開始）與 `DueDateTime`（截止），**無需新增欄位**。前端補上 `datetime-local` 選擇器並可清除。
- **逾期/今天標示**：截止早於現在且未完成＝逾期（紅）；截止或排程為今天＝今天（黃）。清單列左側色條、看板欄位顯示逾期數。

### 3. 子任務（b）→ 新增輕量 `SubTask`「檢核清單」實體
- **決策**：採「檢核清單」設計（標題 + 是否完成 + 排序），而非「子卡片」（不另帶狀態/日期）。理由：符合使用者「列出子任務、看完成進度」的直覺，UI 單純、可算 `已完成/總數` 進度條。
- 新實體 `SubTask`（`AuditableEntity` + `IUserOwned`），對 `(TaskCardId, SortOrder)` 建索引、FK 對 `TaskCard` 為 Cascade。Migration：`AddSubTask`。
- 端點：`GET/POST /api/tasks/{taskId}/subtasks`、`PUT/DELETE /api/subtasks/{id}`、`PUT /api/tasks/{taskId}/subtasks/reorder`。卡片清單 DTO 帶 `SubTaskTotal/SubTaskDone` 供顯示進度。

### 4. 分類 vs 標籤（c）→ 只做「分類」單一維度
- 使用者說「分類 **或** 貼標籤」。**決策：以「分類」（即 repurposed TaskGroup）作為唯一分類維度，不另做標籤系統。** 理由：分類已完整滿足需求；同時上「分類 + 標籤」兩套重疊維度會讓卡片更雜、違反設計原則「選一個方向並貫徹」。若日後需要跨領域標籤，可再加 `Tag`/`TaskTag`（筆記模組已有 `Tag`，可共用）。

### 5. 更新語義：可「清空」日期/分類
- 原 `UpdateTaskCardRequest` 對可空欄位用 `!= null` 判斷，導致「傳 null＝不更新」，**無法清空**。新增 `ClearPlannedDateTime / ClearDueDateTime / ClearGroupId` 三個布林旗標：有值＝設定、`ClearXxx=true`＝清成 null、兩者皆無＝維持原值。如此「拖曳改狀態」這種部分更新不會誤清其他欄位。

### 6. UI/UX（e/f）
- 參考前端設計 skill 與使用者故事，重點：
  - **統一編輯彈窗** `TaskEditorModal`：標題、狀態/優先度分段控制、分類、排程/截止、內容、子任務檢核清單、刪除（二次確認）。清單與看板共用。
  - **快速新增列**：輸入標題按 Enter 即建立（沿用目前分類篩選自動歸類）。
  - **篩選列**：分類 chips（含新增/刪除）＋ 時間（全部/今天/逾期/未排程）。
  - 卡片顯示：核取方塊快速完成、狀態 chip、優先度、分類 chip、截止（逾期紅/今天黃）、子任務進度條。
  - 樣式集中於 `globals.css` 的 `.tk-*`（避免散落 inline、避免 styled-jsx 作用域陷阱）。
