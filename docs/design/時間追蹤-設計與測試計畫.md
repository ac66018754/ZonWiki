# 時間追蹤（Time Tracking）— 設計決策與測試計畫

> 分支：`feature/time-tracking`｜日期：2026-07-15
> 需求來源（使用者原話整理）：記錄每天把時間花在什麼上面。流程：點按鈕打開 ZonWiki 首頁 → 輸入項目名稱 → 可選輸入分類 → 按下儲存（＝開始計時）→ 事情做完再回來按結束 → 計算開始與結束的時間差。因為不一定每次都有時間即時操作，所以要**提供編輯時間**。閱讀可以 **by day / week / month / year** 分組。這塊佔用篇幅較大，要**提供折疊收合**。另外希望**在 iPhone 16 主畫面就能新增項目及結束某項目，不需打開 ZonWiki**。

---

## 1. 背景與關鍵發現（探索結論）

- 本專案是 Minimal API endpoints（非 MVC Controller）：`src/ZonWiki.Api/Endpoints/XxxEndpoints.cs` + `Program.cs` 一行 `MapXxxEndpoints()`。
- Entity 繼承 `AuditableEntity`（六件套稽核欄全在裡面）＋實作 `IUserOwned`（只需 `Guid UserId`）→ 自動吃到「使用者隔離＋ValidFlag」全域查詢過濾；DB 欄位前綴 `{表名}_{欄名}` 由 `ModelBuilderExtensions` 自動套，C# 屬性寫乾淨名字即可。
- **PAT（API 權杖）對端點無感**：`SmartAuthScheme` 依 `Authorization: Bearer` 自動轉發 ApiToken handler，端點照常寫就同時支援 Cookie 與 PAT——iOS 捷徑帶 Bearer 即可呼叫，**不需要另做一套認證**。
- 「分類」現況：無泛用分類表。`Category` 表是筆記階層分類；QuickLink 用**自由文字 `string? Category`**（max 128）。
- 前端：首頁區塊元件放 `frontend/src/components/`（範本：`AiActivitySection.tsx` 骨架＋`QuickLinksSection.tsx` 表單模式）；折疊用「布林 state＋`aria-expanded`＋條件渲染」模式；時區歸日用 `lib/datetime.ts` 的 `formatLocalDate(iso, tz)`（勿用 `.split("T")[0]`）；編輯時間用共用 `DateTimePicker`（UTC ISO 進出）；wall-clock↔UTC 轉換用 `app/tasks/taskUtils.ts` 的 `toLocalInputValue`/`fromLocalInputValue`。

## 2. 決策

| 議題 | 決定 | 理由與取捨 |
|---|---|---|
| 分類的形態 | **自由文字 `string? Category`（max 128）＋既有值 autocomplete（datalist）** | 使用者說「可選輸入分類」——輕量標籤語意，抄 QuickLink 慣例最省事；要升級成共用 Tag/Category 樹再另案（YAGNI）。 |
| 時間差計算 | **不存 Duration 欄位**，DTO 由後端算 `durationSeconds`（進行中為 null） | 存了就有「改時間忘了同步時長」的一致性負擔；算的成本為零。 |
| 進行中的表示 | `EndedDateTime == null` ＝ 計時中 | 單欄位單一真相；不另設狀態欄。 |
| 同時多個計時中 | **允許** | 現實中會同時掛著兩件事（例如「洗衣機」＋「讀書」）；UI 與 stop-latest 都以「最近開始的」為預設對象。 |
| iPhone 主畫面操作 | **iOS 捷徑（Shortcuts）＋ PAT Bearer 呼叫既有端點**；另設 `POST /api/time-entries/stop-latest` 給「一鍵結束」 | iOS Safari 不支援 manifest shortcuts（那是 Android）；捷徑放主畫面＝原生體驗、零 App 開發。stop-latest 讓捷徑不必先列清單再挑。 |
| 編輯結束時間能否「清空回計時中」 | **不能**（PUT 只能改值、不能清成 null） | 避免 Update 請求「未帶 vs 帶 null」語意糾纏；要重新計時就再開一筆（YAGNI）。 |
| ActivityLog | **納入**：`MapEntity` 加 `TimeEntry` case、`FieldLabels` 加欄位中文標籤 | 與全站「活動明細」一致性；成本一個 case。 |
| 策展 OpenAPI（zonwiki-ai.json） | **不加** | 那份是給 ChatGPT Custom GPT 的策展文件；iOS 捷徑不需要 OpenAPI。要接 ChatGPT 再加。 |
| 限流 | 寫入端點掛 `.RequireRateLimiting(RateLimitingExtensions.PatPolicy)`（若實測確認該 policy 依使用者/權杖分區、不誤傷網頁操作） | PAT 外洩時防灌爆，比照 `/api/ai/notes`。 |
| 首頁折疊預設 | 預設**展開**，收合狀態存 localStorage（`zonwiki:time-tracking:collapsed`）；收合時仍顯示標頭（含進行中數量） | 進行中的計時是使用者最需要看見的；記住個人偏好。 |
| 日/週/月/年歸組 | **依使用者時區**歸日（`formatLocalDate`），期間邊界以使用者時區的 wall-clock 換算成 UTC 再給 `from`/`to` 查詢 | 鐵則 #12：DB 存 UTC、前端依時區顯示；跨日項目歸在「開始日」。 |
| **stop-latest 平局 tie-break**（審查補） | `ORDER BY StartedDateTime DESC, CreatedDateTime DESC, Id DESC` | 兩筆 Started 完全相同時（同一秒連按兩次捷徑）行為必須確定性，否則正式環境不可預期、測試 flaky。 |
| **ActivityLog 的時間欄位格式**（審查補，CRITICAL） | `StartedDateTime`/`EndedDateTime` 列入 `FieldLabels`（「開始時間」「結束時間」）但**同時列入 `LongTextFields`（只列欄名、不附前後值）** | 既有 `FormatValue` 對 DateTime 只印 `yyyy-MM-dd`——「同日改時分」會產生前後相同的字串（白紀錄且誤導）；且 Detail 內的值是 UTC，附值會造成時區混淆。只列欄名（「開始時間」）最小、正確、不動既有格式化行為。 |
| **併發控制**（審查補） | **接受 last-write-wins，不加 xmin** | 單人低頻資料，「捷徑結束 vs 網頁編輯」同時發生的機率極低且損失僅一筆時間值（可再編輯救回）；加 409 流程的複雜度不成比例。以測試鎖住此語意。 |
| **垃圾桶**（審查補） | **註冊進 `TrashTypeRegistry`**（軟刪後可在 /trash 還原） | 全站所有 IUserOwned 實體都進垃圾桶；不進會是「刪了找不回」的體驗斷點。成本一個 TypeMap 條目。 |
| **限流的共用桶取捨**（審查補） | 寫入端點掛 `PatPolicy`；**已實查** `ResolveUserPartitionKey` 以 `user:{userId}` 分區、Cookie 與 PAT **共用同一桶**（TokenBucket 30） | 接受：單人系統，手動操作不可能達 30 burst；好處是 PAT 外洩時同樣受限。以 burst 測試鎖住 429 行為。 |
| **PUT 可對進行中項目補上結束時間**（審查補） | 允許（效果等同 /stop，但可自訂時間）；「清空結束時間回計時中」維持不支援（Update 請求 null＝不改，結構上不可表達「清空」） | 使用者「事後補記」的真實路徑；語意單純。 |
| **入參 DateTime 正規化**（審查補，經實查 repo 無既有慣例可比照） | 端點內共用 helper `NormalizeToUtc`：`Utc`→原樣；`Local`（JSON 帶 `+08:00` 這類 offset，STJ 解成 Local）→`ToUniversalTime()`；`Unspecified`（無尾碼）→視為 UTC 補 Kind | iOS 捷徑是本系統第一個會餵「非 Z 尾碼」時間字串的用戶端；不正規化會 500（Npgsql timestamptz 要求 Utc Kind）。**技術債留痕**：既有 `TaskEndpoints` 對 `DueDateTime` 等同樣沒有正規化（僅靠前端一律送 Z 的不變式撐著），本次不修、記入 DECISIONS.md。 |

## 3. 資料模型（新表 `TimeEntry`）

`src/ZonWiki.Domain/Entities/TimeEntry.cs`：

```csharp
public class TimeEntry : AuditableEntity, IUserOwned
{
    public Guid UserId { get; set; }
    public string Title { get; set; } = string.Empty;   // 項目名稱，必填，max 200
    public string? Category { get; set; }               // 自由文字分類，max 128，可空
    public DateTime StartedDateTime { get; set; }        // 開始時間（UTC）
    public DateTime? EndedDateTime { get; set; }         // 結束時間（UTC）；null = 計時中
}
```

- Configuration：`HasMaxLength(200)`（Title, required）、`HasMaxLength(128)`（Category）、索引 `(UserId, StartedDateTime)`（日週月年區間查詢）、`(UserId, EndedDateTime)`（撈進行中）。
- Migration：`AddTimeEntry`（`dotnet ef migrations add AddTimeEntry --project src/ZonWiki.Infrastructure --startup-project src/ZonWiki.Api`），啟動自動套用。
- DB 欄位自動成 `TimeEntry_Id`、`TimeEntry_Title`、`TimeEntry_StartedDateTime`…（符合全域命名鐵則）。

## 4. 後端 API（`src/ZonWiki.Api/Endpoints/TimeEntryEndpoints.cs`）

DTO 放 `src/ZonWiki.Domain/Dtos/TimeEntryDtos.cs`（sealed record）：

```
TimeEntryDto(Id, Title, Category, StartedDateTime, EndedDateTime, DurationSeconds)
CreateTimeEntryRequest(Title, Category?, StartedDateTime?)      // Started 未帶 = UtcNow
StopTimeEntryRequest(EndedDateTime?)                            // 未帶 body 或未帶欄位 = UtcNow
UpdateTimeEntryRequest(Title?, Category?, StartedDateTime?, EndedDateTime?)  // null = 不改；Category 空字串 = 清為未分類
```

| 端點 | 行為 | 驗證／錯誤 |
|---|---|---|
| `GET /api/time-entries?from=&to=` | 列出 `StartedDateTime ∈ [from, to)` 的項目（含已結束與進行中），依 StartedDateTime 逆序 | from/to 必填且 from < to → 否則 400 |
| `GET /api/time-entries/running` | 列出所有進行中（EndedDateTime == null），依 StartedDateTime 逆序 | — |
| `GET /api/time-entries/categories` | 本人用過的 distinct 非空分類（供 autocomplete） | — |
| `POST /api/time-entries` | 建立並開始計時；Started 未帶＝UtcNow | Title 空白→400；Title>200→400；Category>128→400；若帶 EndedDateTime 概念——不支援（Create 不收 Ended） |
| `POST /api/time-entries/{id}/stop` | 結束計時；Ended 未帶＝UtcNow | 不存在/非本人→404；已結束→400；Ended < Started→400 |
| `POST /api/time-entries/stop-latest` | 結束「最近開始的進行中項目」（iOS 捷徑一鍵結束） | 無進行中→404；回傳被結束的項目（含時長，供捷徑通知顯示） |
| `PUT /api/time-entries/{id}` | 編輯名稱/分類/開始/結束時間 | 404 同上；改完後若兩端皆有值且 Ended < Started→400；Title 給了但空白→400 |
| `DELETE /api/time-entries/{id}` | 軟刪除（ValidFlag=false + DeletedDateTime），回 204 | 404 同上 |

- 所有 DateTime 入參一律正規化成 UTC（`Kind==Unspecified` 視為 UTC 補 Kind；`Local` 轉 UTC）——Npgsql timestamptz 要求 UTC Kind（實作時比照既有端點的處理慣例）。
- 取 UserId 慣例：`httpContext.User.FindFirst(AuthExtensions.UserIdClaimType)`（Cookie 與 PAT 皆會塞此 claim）。
- 回應信封一律 `ApiResponse<T>`。

## 7. 測試計畫（**已經 sub-agent 對抗式審查並依結論修訂**，審查結果見 §7.9）

測試放 `tests/ZonWiki.Api.Tests/Integration/TimeEntryEndpointsHttpTests.cs`（走真 HTTP＋Testcontainers 真 PostgreSQL＋真驗證，`[Collection(ApiIntegrationCollection.Name)]`，比照 `ActivityLogDetailHttpTests` 的基座用法）。理由：本功能核心風險在「端點驗證邏輯＋隔離＋PAT 路徑」，HTTP 整合測試才測得到（InMemory 直打 DbContext 測不到端點行為）。

**通用不變式（審查補）**：
- **每個測試方法各自 `SeedUserWithTokenAsync` 建獨立使用者**（獨立 GUID email），不得共用 client——共用 DB 容器下靠使用者隔離防測試互染（比照 `ActivityLogDetailHttpTests`）。
- **每個測試方法與 helper 都要繁體中文 XML `<summary>`**（鐵則 #5/#6，比照既有測試檔無一例外）。
- **排序敏感測試一律顯式指定 `StartedDateTime`**，不依賴「兩次 HTTP 呼叫的自然時間差」。
- **「預設 now」類斷言**：在呼叫前後各取 `DateTime.UtcNow` 夾出區間、斷言值落在區間內（容差＝呼叫本身耗時，不設魔術容差值）；**時長計算類斷言**：兩端都用測試指定的明確 UTC 時間戳，斷言精確相等（零 flaky）。

### 7.1 建立（Create）
1. `Create_DefaultsStartToUtcNow_AndIsRunning`：不帶 Started → 201；startedDateTime 落在呼叫前後夾出的區間、endedDateTime null、durationSeconds null。
2. `Create_WithExplicitStart_StoresGivenTime`：帶 `2026-07-01T02:00:00Z` → 回存相同 UTC 時刻。
3. `Create_WithUnspecifiedKindStart_TreatedAsUtc`：帶無時區尾碼的 `2026-07-01T02:00:00` → 不噴 500，存為 UTC 同值。
3b. `Create_WithOffsetStart_ConvertsToUtc`（審查補，iOS 捷徑真實格式）：帶 `2026-07-15T14:30:00+08:00` → 存為 `2026-07-15T06:30:00Z`。
4. `Create_EmptyTitle_Returns400`；`Create_TitleOver200_Returns400`；`Create_CategoryOver128_Returns400`。
4b. `Create_TitleExactly200_CategoryExactly128_Succeeds`（審查補，邊界合法側）。
5. `Create_TrimsTitleAndCategory_EmptyCategoryStoredAsNull`：`"  讀書  "`/`"  "` → Title 修剪、Category 為 null。

### 7.2 結束（Stop / Stop-latest）
6a. `Stop_WithoutBody_DefaultsEndToUtcNow`：不帶 body → 200、endedDateTime 落在呼叫前後區間（審查補：與時長計算拆開）。
6b. `Stop_DurationComputation_IsExact`：Started/Ended 皆測試指定（相差 90 分）→ durationSeconds == 5400（精確，零 flaky）。
7. `Stop_WithExplicitEnd_UsesGivenTime`。
8. `Stop_EndBeforeStart_Returns400`。
8b. `Stop_EndEqualsStart_Returns200WithZeroDuration`（審查補，零時長邊界合法）。
9. `Stop_AlreadyStopped_Returns400`。
10. `Stop_NotFound_Returns404`；`Stop_OtherUsersEntry_Returns404`（隔離）。
11. `StopLatest_StopsMostRecentlyStartedRunning`：兩筆進行中（Started 顯式相差 1 小時）→ 結束較晚那筆，另一筆仍在跑。
11b. `StopLatest_TiedStartTimes_UsesDeterministicTiebreak`（審查補）：兩筆 Started 完全相同、CreatedDateTime 不同（DB scope 種子）→ 結束 CreatedDateTime 較晚那筆（tie-break 確定性）。
11c. `StopLatest_OnlyConsidersOwnRunningEntries`（審查補）：B 有「更晚開始」的進行中項目，A 呼叫 stop-latest → 結束的是 A 自己的、B 的不受影響。
12. `StopLatest_NoRunning_Returns404`。

### 7.3 查詢（List / Running / Categories）
13. `List_FiltersByStartedRange_InclusiveFrom_ExclusiveTo`（審查補：五點邊界）：`from-1s`（排除）、`== from`（含）、`to-1s`（含）、`== to`（排除）、`to+1s`（排除）。
13b. `List_LongRunningEntryStartedBeforeRange_ExcludedFromList_ButVisibleViaRunning`（審查補：鎖住「歸開始日」取捨）。
14. `List_MissingOrInvalidRange_Returns400`（缺 from/to、from ≥ to）。
14b. `List_MalformedDateQueryString_Returns400NotServerError`（審查補：`from=abc` 走 binding 失敗路徑）。
15. `List_OnlyReturnsOwnEntries`（隔離）。
16. `Running_ReturnsOnlyRunning_OrderedByStartDesc_OwnOnly`（審查補：同時斷言跨使用者隔離；Started 顯式指定）。
17. `Categories_ReturnsDistinctNonNull_OwnOnly`。

### 7.4 編輯／刪除（Update / Delete）
18. `Update_ChangesTitleCategoryStartEnd`：全欄位改 → 回傳與 DB 皆更新；durationSeconds 重新計算（用明確時間戳斷言精確值）。
19. `Update_PartialNullFieldsUnchanged`：只帶 Title → 其餘不變。
20. `Update_CategoryEmptyOrWhitespace_ClearsToNull`（審查補：`""` 與 `"   "` 皆清為 null，對齊 QuickLink 的 `IsNullOrWhiteSpace` 語意）。
21a. `Update_NewStartAfterExistingEnd_Returns400`（審查補：交叉方向一）。
21b. `Update_NewEndBeforeExistingStart_Returns400`（審查補：交叉方向二）。
21c. `Update_EndEqualsStart_Allowed`（審查補：零時長合法）。
21d. `Update_SetsEndOnRunningEntry_StopsIt`（審查補：PUT 補結束時間＝合法的事後補記路徑）。
21e. `Update_AfterStop_OverwritesWithoutConflict`（審查補：鎖住 last-write-wins 決策——無版本檢查、後寫贏）。
22. `Update_BlankTitle_Returns400`；`Update_TitleOver200_Returns400`；`Update_CategoryOver128_Returns400`（審查補：Update 側長度驗證）；`Update_NotFound_Returns404`；`Update_OtherUsers_Returns404`。
23. `Delete_SoftDeletes_EntryGoneFromListButRowRemains`：204 → List 不見；`IgnoreQueryFilters` 仍在、ValidFlag=false（**不可硬刪**鐵則）。
23b. `Delete_OtherUsersEntry_Returns404`（審查補：Delete 側隔離）。
23c. `Delete_RunningEntry_SoftDeletesSuccessfully`（審查補）。
23d. `Delete_TimeEntry_AppearsInTrashAndCanBeRestored`（審查補：垃圾桶決策的鎖定測試）。

### 7.5 PAT（iOS 捷徑路徑）＋限流
24. `PatBearer_CanCreateAndStopLatest`：帶 Bearer → 建立＋stop-latest 全流程（等同 iPhone 捷徑實際走法）。
25. `NoAuth_Returns401`（FallbackPolicy 防線）。
