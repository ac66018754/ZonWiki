## 2. 實作詳細說明

### 2.1 筆記 CRUD 端點

#### POST /api/notes - 建立筆記

**功能**：
- 驗證必填欄位（Title、ContentRaw）
- 自動產生 slug（去除特殊字元、轉小寫、連字號分隔）
- 檢查同使用者內 slug 唯一性
- 用 Markdig 渲染 HTML（禁用 raw HTML 防 XSS）
- 計算 ContentHash（SHA-256）
- 自動建立 NoteRevision（RevisionNo=1, ChangeKind="create"）
- 支援指派分類、標籤
- 自動解析 Wiki 連結並建立 NoteLink

**輸入**：
```json
{
  "title": "My Note",
  "contentRaw": "# Title\n\nContent here",
  "kind": "note",
  "isDraft": false,
  "categoryIds": ["uuid1", "uuid2"],
  "tagIds": ["uuid3", "uuid4"]
}
```

**輸出**：
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "title": "My Note",
    "slug": "my-note",
    "contentHtml": "<h1>Title</h1>\n<p>Content here</p>",
    "contentRaw": "# Title\n\nContent here",
    "kind": "note",
    "isDraft": false,
    "createdDateTime": "2026-06-11T03:20:00Z",
    "updatedDateTime": "2026-06-11T03:20:00Z",
    "commentCount": 0
  }
}
```

#### PUT /api/notes/{id} - 更新筆記

**功能**：
- 部份更新（title、contentRaw、isDraft 均可空）
- 重新渲染 HTML（若內容變更）
- 重新計算 ContentHash
- 遞增版本序號、記錄 ChangeKind="update"
- 支援部份更新分類、標籤
- 若內容變更，重新解析 Wiki 連結

#### DELETE /api/notes/{id} - 軟刪除筆記

**功能**：
- 軟刪除（ValidFlag=false + DeletedDateTime）
- 記錄 ChangeKind="delete" 的版本

### 2.2 版本控制 / 編輯歷史

### 2.3 多對多 關係管理

#### PUT /api/notes/{id}/categories - 指派分類

**輸入**：分類 UUID 清單  
**功能**：覆寫現有分類（軟刪除舊關聯、建立新關聯）

#### PUT /api/notes/{id}/tags - 指派標籤

**輸入**：標籤 UUID 清單  
**功能**：覆寫現有標籤；標籤不存在時自動建立（TODO P5 改為傳標籤名稱）

### 2.4 Wiki 連結 / 知識圖譜

#### GET /api/notes/{id}/backlinks - 反向連結

**功能**：列出所有指向此筆記的連結

**輸出**：
```json
{
  "success": true,
  "data": [
    {
      "id": "uuid",
      "sourceNoteId": "uuid",
      "sourceNoteTitle": "Source Note",
      "sourceNoteSlug": "source-note",
      "anchorText": "My Note"
    }
  ]
}
```

#### GET /api/graph - 知識圖譜

**功能**：回傳所有筆記與連結，供前端繪製圖譜（若驗證啟用則過濾為使用者自己的）

**輸出**：
```json
{
  "success": true,
  "data": {
    "nodes": [
      {
        "id": "uuid",
        "title": "Note Title",
        "slug": "note-slug",
        "kind": "note"
      }
    ],
    "edges": [
      {
        "sourceNoteId": "uuid",
        "targetNoteId": "uuid",
        "anchorText": "referenced note"
      }
    ]
  }
}
```

## 4. 技術決策與實作細節

### 4.1 Markdown 渲染

- **引擎**：Markdig（已在 NotesSyncService 中使用）
- **配置**：禁用 raw HTML 防 XSS
- **管線**：`.UseAdvancedExtensions().UseAutoLinks().DisableHtml()`

### 4.2 Wiki 連結解析

- **格式**：`[[AnchorText]]`（regex：`\[\[([^\]\r\n]+)\]\]`）
- **目標尋找**：優先 slug 相符，次優 title 完全相符
- **未命中**：允許 TargetNoteId=null（待建立的連結）

### 4.3 ContentHash

- **演算法**：SHA-256（UTF-8 編碼）
- **用途**：匯入時判斷檔案是否有變、衝突偵測
- **輸出格式**：64 字元十六進位字串

### 4.4 使用者隔離

- **機制**：每個端點從 claim 或查詢參數取 userId
- **過濾**：WHERE 子句 `n.UserId == userId && n.ValidFlag`
- **開發用**：支援 `?userId=uuid` 查詢參數（P6 改為完全授權)

### 4.5 AI 服務架構

**介面設計**：
```csharp
public interface INoteAiService
{
    Task<string> ReformatAsync(string contentRaw, CancellationToken ct);
    Task<string> BeautifyAsync(string contentRaw, CancellationToken ct);
}
```

**DI 註冊**：
