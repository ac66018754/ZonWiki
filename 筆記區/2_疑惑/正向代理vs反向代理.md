# 正向代理 vs 反向代理

> 在 Docker 工作坊中，Nginx 被稱為「反向代理」，那反向是什麼意思？有正向代理嗎？

---

## 一句話區分

- **正向代理**：幫「客戶端」出去拿東西
- **反向代理**：幫「伺服器」接收請求

---

## 正向代理（Forward Proxy）

```
你的電腦 → 正向代理 → 網際網路
```

你不直接連目標網站，而是透過代理幫你連。**目標網站不知道真正的你是誰。**

### 日常例子

- **VPN** — 你連 VPN，VPN 幫你去拿國外的網頁
- **公司網路的 Proxy** — 公司所有人的請求都經過代理，代理可以擋掉特定網站、記錄瀏覽紀錄

### 日常比喻

你請朋友幫你去買東西，店家只看到你朋友，不知道東西其實是你要的。

---

## 反向代理（Reverse Proxy）

```
使用者 → 反向代理 → 後端伺服器
```

使用者不直接連後端，而是先到反向代理，由它決定要轉給誰。**使用者不知道後面真正的伺服器是誰。**

### Docker 工作坊的例子

```
使用者打 http://localhost
    ↓
  Nginx（反向代理）
    ├── /api/*  → 轉給後端
    └── /*      → 轉給前端
```

使用者只看到一個入口（`http://localhost`），不知道後面其實有前端和後端兩個服務。

### 日常比喻

餐廳的前台接待，客人只跟前台講話，前台再把需求分配給不同的廚房。客人不需要知道有幾個廚房、哪個廚師在做菜。

---

## 對比

| | 正向代理 | 反向代理 |
|--|---------|---------|
| **幫誰** | 幫客戶端（你） | 幫伺服器 |
| **誰不知道真相** | 目標網站不知道你是誰 | 使用者不知道後面有幾台伺服器 |
| **常見用途** | VPN、翻牆、公司網路管控 | 分流請求、負載平衡、隱藏後端 |
| **常見軟體** | Shadowsocks、Squid、V2Ray | Nginx、Caddy、Traefik |

---

## 為什麼叫「反向」？

因為代理的方向反過來了：

- **正向**：代理站在**客戶端**這邊，幫你往外送請求
- **反向**：代理站在**伺服器**這邊，幫伺服器往內收請求
