# 環境變數：後端工程師的深度指南

作為一名後端工程師，掌握環境變數（Environment Variables）就像是掌握了作業系統層級的「設定檔（Config）」。它能讓你的程式碼與環境解耦（Decoupling），是實踐 [Twelve-Factor App](https://12factor.net/config) 開發原則的核心技術之一。以下為你整理的深度知識與進階用法：

## 一、環境變數的三大層級

在 Windows 中，環境變數並非只有一種，它們依據「影響範圍」由小到大分為：

| 層級 (Scope) | 說明                                       | 生命週期                     |
| :----------- | :----------------------------------------- | :--------------------------- |
| Process (行程) | 僅存在於目前的視窗或執行的程式中。         | 視窗關閉或程式結束即消失。   |
| User (使用者)  | 只對目前登入的 Windows 帳號有效。          | 永久儲存於登錄檔 (Registry)。|
| Machine (系統) | 對這台電腦的所有使用者、所有服務（含 IIS, Docker）都有效。 | 永久儲存，修改需管理員權限。 |

## 二、為什麼後端工程師需要它？

### 1. 安全性 (Secrets Management)

如你所見，我們絕不把 API Key、資料庫密碼寫在 `appsettings.json` 或程式碼中。透過環境變數，你可以讓程式在執行時動態讀取：

- **本地開發**：讀取你電腦的 `GEMINI_API_KEY`。
- **CI/CD (GitHub Actions / GitLab CI)**：在 Pipeline 中注入變數。
- **正式環境 (Azure/AWS/K8s)**：透過容器配置注入密碼。

### 2. 環境切換 (Env Switching)

你可以設定一個 `ASPNETCORE_ENVIRONMENT` 變數：

- 在你電腦設為 `Development`。
- 在伺服器設為 `Production`。

程式碼只需寫 `if (env.IsDevelopment())` 即可自動切換不同的邏輯。

## 三、查看環境變數

### PowerShell

```powershell
# 查看所有環境變數
Get-ChildItem Env:

# 尋找特定變數（例如搜尋跟 Git 相關的）
Get-ChildItem Env: | Where-Object { $_.Name -like "*GIT*" }
```

## 四、C# 如何取得環境變數

在 C# 中，你可以透過 `Environment` API 或 `IConfiguration` 抽象層來取得環境變數。推薦使用 `IConfiguration`，它提供了更強大、更靈活的配置管理能力。

| 特性       | `Environment` API          | `IConfiguration` 抽象層                               |
| :--------- | :------------------------- | :---------------------------------------------------- |
| **資料來源** | 僅限環境變數               | JSON, XML, 環境變數, 命令列參數, Memory, Key Vault 等 |
| **分層複寫** | 無                         | 有 (Priority)。後加入的 Provider 會覆蓋掉前面的值    |
| **結構化** | 僅能讀取單一字串           | 支援 Options Pattern，可直接綁定到 C# Class 物件    |
| **動態更新** | 修改後通常需重啟           | 支援 `IOptionsSnapshot`，檔案變動可即時反應 (Hot Reload) |
| **測試友善度** | 難以 Unit Test (需模擬全域狀態) | 極易測試 (可 Mock `IConfiguration` 物件)              |

### 範例：使用 `IConfiguration` 綁定到 Class

```csharp
public class GeminiOptions
{
    public string ApiKey { get; set; }
    public double Temperature { get; set; }
}

// 在 Startup.cs 或 Program.cs 中配置
builder.Services.Configure<GeminiOptions>(builder.Configuration.GetSection("Gemini"));
```

### 範例：直接從 `IConfiguration` 讀取

```csharp
public class MyService(IConfiguration config)
{
    private readonly IConfiguration _config = config;

    public void DoSomething()
    {
        // 直接讀取，框架會自動從 appsettings.json 或環境變數中尋找
        var key = _config["GEMINI_API_KEY"]; 
    }
}
```

### 範例：使用 `Environment` API (不推薦用於應用程式配置)

```csharp
string apiKey = Environment.GetEnvironmentVariable("GEMINI_API_KEY");
```
